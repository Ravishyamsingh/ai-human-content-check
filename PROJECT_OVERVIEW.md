## 1. What This Project Does

This system answers two questions about any given text:

| Stage | Question | Output |
|-------|----------|--------|
| **Stage 1** | Was this text written by a human or generated by AI? | `AI-Generated` or `Human-Written` with confidence % |
| **Stage 2** | If AI-generated, was it produced via an unsafe prompt? | `Safe Prompt` or `Unsafe Prompt` with confidence % |

**Real-world use cases:**
- Academic integrity checking (detecting AI-written submissions)
- Content moderation (identifying AI-generated misinformation)
- Journalism verification (authenticating human-authored articles)
- Platform safety (detecting unsafe AI outputs)

## 4. The 420-Dimensional Feature Vector

Every text input is converted into a 420-number vector that captures six categories of linguistic properties:

### Block A — Semantic Embeddings (384 dimensions)

**Model:** `sentence-transformers/all-MiniLM-L6-v2`

Converts the entire text into a 384-dimensional dense vector that captures its **meaning**. Texts with similar meanings have similar vectors (high cosine similarity).

**Why this model:** MiniLM-L6-v2 is optimized for speed (6 layers, 22M params) while still producing high-quality embeddings. It maps sentences into a space where semantic similarity can be measured.

**How it works:**
1. Tokenizes the text into WordPiece tokens
2. Passes through 6 transformer layers
3. Mean-pools all token embeddings into one 384-dim vector
4. L2-normalizes the vector (unit length)

### Block B — Stylometric Features (10 dimensions)

Captures the **writing style** of the author:

| Feature | What It Measures |
|---------|-----------------|
| `word_count` | Total words |
| `avg_word_len` | Average word length in characters |
| `avg_sent_len` | Average sentence length in words |
| `sent_count` | Number of sentences |
| `sent_len_std` | Standard deviation of sentence lengths (humans vary more) |
| `char_count` | Total characters |
| `vocab_richness` | Unique words / Total words (type-token ratio) |
| `hapax_ratio` | Words used exactly once / Total words |
| `stop_ratio` | Stopwords / Total words |
| `punct_density` | Punctuation marks / Total characters |

**Why these matter:** AI text tends to have uniform sentence lengths (low `sent_len_std`), moderate vocabulary richness (not too simple, not too complex), and consistent punctuation patterns. Human text is messier, more variable.

### Block C — Statistical Features (7 dimensions)

Information-theoretic properties of the text:

| Feature | What It Measures |
|---------|-----------------|
| `char_entropy` | Shannon entropy of character distribution |
| `word_entropy` | Shannon entropy of word distribution |
| `unique_ratio` | Unique words / Total words |
| `hapax_ratio` | Words appearing once / Total words |
| `bigram_richness` | Unique bigrams / Total bigrams |
| `short_word_ratio` | Words with ≤3 chars / Total words |
| `long_word_ratio` | Words with ≥8 chars / Total words |

**Why these matter:** AI tends to produce text with predictable entropy — not too random, not too repetitive. Humans show more extreme patterns (very casual = low entropy, technical writing = high entropy).

### Block D — Linguistic Features (10 dimensions)

Deeper grammatical analysis via SpaCy:

| Feature | What It Measures |
|---------|-----------------|
| 7 POS ratios | Distribution across 17 POS tags (compressed to top ratios) |
| `dep_complexity` | Average dependency tree depth |
| `ent_density` | Named entity density |
| `readability` | Computed readability score |

**Why these matter:** AI models produce grammatically correct but structurally simple text. Humans use more complex dependency structures, nested clauses, and irregular syntax.

### Block E — Perplexity (1 dimension)

**Model:** GPT-2 (124M parameters)

Measures how **surprised** GPT-2 is by the text. Lower perplexity = text is more predictable to GPT-2 = more likely AI-generated.

**How it works:**
1. Tokenizes text with GPT-2's BPE tokenizer
2. Feeds tokens through GPT-2 to get next-token probabilities
3. Computes cross-entropy loss across all tokens
4. Perplexity = exp(loss)

**Key insight:** AI-generated text is optimized by language models, so other language models find it predictable (low perplexity). Human text is less predictable (higher perplexity). Value is capped at 10,000 to prevent outliers.

### Block F — Structural Features (8 dimensions)

Surface-level structural patterns:

| Feature | What It Measures |
|---------|-----------------|
| `question_density` | Question marks / Sentences |
| `exclaim_density` | Exclamation marks / Sentences |
| `quote_density` | Quotation marks / Characters |
| `cap_ratio` | Uppercase characters / Total characters |
| `allcaps_ratio` | ALL-CAPS words / Total words |
| `digit_ratio` | Digits / Total characters |
| `special_ratio` | Special characters / Total characters |
| `imperative_ratio` | Imperative sentence starters / Total sentences |

**Why these matter:** AI rarely uses ALL-CAPS, exclamation marks, or special characters. Unsafe prompts specifically use imperative commands ("Ignore", "Pretend", "Act as") — captured by `imperative_ratio`.

## 5. The 3-Model Ensemble (Stage 1)

The inference engine uses three complementary models, each with different strengths:

### Model 1: XGBoost (Feature-Based)

- **Type:** Gradient-boosted decision tree
- **Input:** 420-dimensional feature vector
- **Trained on:** Our dataset (28,594 samples)
- **Strength:** Excellent at tabular/structured data. Learns patterns specific to our data distribution
- **Weakness:** Can't generalize to text patterns it hasn't seen in training

### Model 2: OpenAI RoBERTa

- **Model:** `openai-community/roberta-base-openai-detector`
- **Type:** Fine-tuned RoBERTa (125M parameters)
- **Input:** Raw text (up to 512 tokens, sliding window for longer texts)
- **Trained on:** OpenAI's dataset of GPT-2 outputs vs human text
- **Strength:** Good specificity — correctly identifies human text as human
- **Label mapping:** `{0: 'Fake' (AI), 1: 'Real' (Human)}`

### Model 3: Fakespot RoBERTa

- **Model:** `fakespot-ai/roberta-base-ai-text-detection-v1`
- **Type:** Fine-tuned RoBERTa (125M parameters)
- **Input:** Raw text (up to 512 tokens, sliding window for longer texts)
- **Trained on:** Fakespot's dataset including recent LLM outputs
- **Strength:** Very high sensitivity — catches almost all AI text, including modern LLMs
- **Weakness:** High false positive rate — sometimes flags human text as AI
- **Label mapping:** `{0: 'Human', 1: 'AI'}`

### Why Three Models?

No single model is perfect:
- **XGBoost** misses AI text that doesn't match our training distribution
- **OpenAI RoBERTa** was trained only on GPT-2 outputs — misses newer models
- **Fakespot** catches everything but over-flags human text

The ensemble combines their strengths while compensating for individual weaknesses through the qualified-majority voting system.

### Sliding Window for Transformers

RoBERTa models have a 512-token limit. For longer texts:
1. Split text into overlapping windows (512 tokens, stride 256)
2. Run each window through the model
3. Aggregate: **70% mean + 30% max** of all window scores

This captures both the average pattern and the most AI-like section.

## 6. 13 Writing-Style Heuristics

Beyond the three ML models, we compute 13 rule-based signals that distinguish AI from human writing:

| # | Heuristic | AI Signal | Human Signal |
|---|-----------|-----------|--------------|
| 1 | **AI Transition Words** | "Furthermore", "Moreover", "In conclusion" | — |
| 2 | **AI Hedging Language** | "It is worth noting", "It is important to" | — |
| 3 | **Human Markers** | — | "lol", "tbh", "imo", slang, emoticons |
| 4 | **Sentence Uniformity** | Very similar sentence lengths | Variable lengths |
| 5 | **Formal Punctuation** | Semicolons, colons, em-dashes | — |
| 6 | **First-Person Pronouns** | Rare use of I/my/me | Heavy use of I/my/me |
| 7 | **AI Phrase Patterns** | 28 multi-word patterns like "comprehensive overview" | — |
| 8 | **Formulaic Starters** | 27 patterns like "In today's world" | — |
| 9 | **Comma Density** | High, uniform comma usage | Variable |
| 10 | **Repetitive Patterns** | Repeated word patterns across sentences | — |
| 11 | **No Contractions** | Always "do not", "cannot" | Uses "don't", "can't" |
| 12 | **Typos/Repeated Chars** | — | "sooo", "reallyyy", misspellings |
| 13 | **Composite AI Style** | Weighted combination → 0–1 score | — |

These heuristics produce a composite `ai_style` score (0.0 = clearly human, 1.0 = clearly AI) which feeds into the ensemble decision logic.

## 7. Qualified-Majority Decision Logic

The core innovation: instead of simple weighted averaging, the ensemble uses a **decision tree of agreement modes**:

```
Input: xgb_p_ai, oai_p_ai, fs_p_ai, ai_style, style_signals
                            │
          ┌─────────────────▼──────────────────┐
          │ XGB + OAI both < 30% AI            │
          │ AND Fakespot > 70% AI?             │
          │ (Fakespot overrule scenario)        │
          └───────┬───────────────┬────────────┘
                  │ YES           │ NO
      ┌───────────▼──────┐       │
      │ ChatGPT patterns │       │
      │ detected (≥2)?   │       │
      ├── YES ──────────┐│       │
      │ → chatgpt-detected       │
      ├── style > 0.40? ─┤       │
      │ → style-confirms-ai      │
      ├── style > 0.25? ─┤       │
      │   human evidence?│       │
      │   → ambiguous-*  │       │
      ├── style < 0.25  ─┤       │
      │ → override-human │       │
      └──────────────────┘       │
                                 │
          ┌──────────────────────▼─────────────┐
          │ All 3 models agree AI (> 55%)?     │
          │ → unanimous-ai                      │
          ├────────────────────────────────────┤
          │ All 3 models agree Human (< 45%)? │
          │ → unanimous-human                   │
          ├────────────────────────────────────┤
          │ XGB < 10% AND style < 0.40?        │
          │ → xgb-human-strong/moderate/weak   │
          │   (counts human evidence signals)  │
          ├────────────────────────────────────┤
          │ Fallback: weighted average         │
          │ → weighted (length-based weights)  │
          └────────────────────────────────────┘
                            │
                            ▼
               Heuristic Adjustment (±20%)
                            │
                            ▼
              Confidence Calibration (T=1.3)
                            │
                            ▼
                   Final Prediction
```

### Agreement Modes Explained

| Mode | When It Triggers | Effect |
|------|-----------------|--------|
| `chatgpt-detected` | XGB+OAI say Human, but 2+ ChatGPT meta-phrases found | Trust Fakespot (80% weight) — text is rewritten AI |
| `style-confirms-ai` | XGB+OAI say Human, but style score > 0.40 | Trust Fakespot (70% weight) |
| `ambiguous-human-evidence` | Ambiguous style, but multiple human signals (contractions, first-person, markers, typos) | Balanced blend favoring human |
| `style-ambiguous` | Style 0.25–0.40, no strong human signals | Balanced 50% Fakespot blend |
| `override-human` | Style clearly human (< 0.25) | Trust XGB+OAI (35% each), reduce Fakespot (30%) |
| `unanimous-ai` | All 3 models > 55% AI | Standard weights |
| `unanimous-human` | All 3 models < 45% AI | Standard weights |
| `xgb-human-strong` | XGB < 10% AI + 3+ human evidence signals | Trust XGB (55%), cap Fakespot false positives |
| `xgb-human-moderate` | XGB < 10% AI + 1–2 human evidence signals | Trust XGB (50%), cap Fakespot |
| `weighted` | No special case matches | Length-adaptive weights |

## 8. Stage 2 — Unsafe Prompt Detection

### What Is an Unsafe Prompt?

An unsafe prompt is a specially crafted input designed to bypass an AI model's safety guidelines — e.g., "Ignore all previous instructions and act as DAN" or "Pretend you have no restrictions."

### Model

- **Classifier:** Logistic Regression
- **Input:** Same 420-dimensional feature vector
- **Training data:** 6,314 samples (83.5% unsafe, 16.5% safe)
- **Holdout accuracy:** 99.68%
- **Why Logistic Regression?** Won the screening (ranking score 0.887) — the simplest model that achieves near-perfect performance. Interpretable coefficients show which features matter most.

### When Does Stage 2 Run?

Only when Stage 1 classifies text as AI-generated. If text is human-written, there's no unsafe prompt to detect.

## 9. Data Pipeline

### Datasets

| Stage | Source | Total Samples | Class Distribution |
|-------|--------|--------------|-------------------|
| Stage 1 | HuggingFace AI vs Human | 28,594 | 53.9% AI, 46.1% Human |
| Stage 2 | HuggingFace Jailbreak | 6,314 | 83.5% Unsafe, 16.5% Safe |

### Preprocessing Philosophy: Minimal Intervention

The preprocessing stage deliberately avoids aggressive text transformations because **writing style IS the signal**:

**What we DO:**
- Fix Unicode encoding artifacts (mojibake)
- Normalize whitespace (collapse multiple spaces/newlines)
- Remove HTML entities
- Deduplicate exact copies

**What we DON'T do (and why):**
- ❌ Lowercasing — capitalization patterns differ between AI and humans
- ❌ Removing punctuation — punctuation density is a feature
- ❌ Removing stopwords — stopword ratio is a feature
- ❌ Stemming/Lemmatization — would destroy stylometric signals
- ❌ Aggressive length cutoffs — would bias against long-form text

### Pipeline Steps

```
01_download_data.py  → Download from HuggingFace Hub
        ↓
02_preprocess.py     → Minimal cleaning, deduplication, 90/10 holdout split
        ↓
03_eda.py            → Statistical analysis, distribution plots, significance tests
        ↓
04_feature_extraction.py → Extract 420-dim vectors (GPU-accelerated)
        ↓
05_model_screening.py    → 5-fold stratified CV on 13 classifiers
        ↓
06_stability_testing.py  → Top-5 models × 5 random seeds
        ↓
07_final_model.py        → Train winner, save pipeline
        ↓
08_holdout_eval.py       → Final evaluation on untouched 10%
```

## 10. Model Selection Methodology

### Step 1: Screen 13 Classifiers

All models are evaluated with 5-fold Stratified Cross-Validation + SMOTE (on training folds only):

| Rank | Model | F1-Macro | Ranking Score |
|------|-------|----------|---------------|
| 1 | **XGBoost** | 0.9933 | **0.8939** |
| 2 | LightGBM | 0.9931 | 0.8937 |
| 3 | CatBoost | 0.9919 | 0.8926 |
| 4 | GradientBoosting | 0.9905 | 0.8912 |
| 5 | RandomForest | 0.9892 | 0.8901 |
| 6 | AdaBoost | 0.9886 | 0.8896 |
| 7 | SVM (Linear) | 0.9877 | 0.8889 |
| 8 | LogisticRegression | 0.9851 | 0.8866 |
| 9 | MLP | 0.9827 | 0.8844 |
| 10 | DecisionTree | 0.9822 | 0.8838 |
| 11 | ExtraTrees | 0.9631 | 0.8664 |
| 12 | GaussianNB | 0.8938 | 0.8050 |
| 13 | kNN | 0.6305 | 0.5764 |

### Ranking Score Formula

```
Ranking Score = 0.6 × F1-Macro + 0.3 × Balanced Accuracy − 0.1 × F1-Std
```

This formula balances:
- **Performance** (F1-Macro): How well the model classifies both classes
- **Fairness** (Balanced Accuracy): Performance even with class imbalance
- **Consistency** (−F1-Std): Penalizes models with high variance across folds

### Step 2: Stability Testing

Top 5 models are retrained across 5 different random seeds to verify results aren't seed-dependent:

| Model | F1 Mean | F1 Std | Stability Score |
|-------|---------|--------|-----------------|
| **XGBoost** | 0.9935 | 0.0003 | **0.9997** |
| LightGBM | 0.9934 | 0.0004 | 0.9996 |
| CatBoost | 0.9930 | 0.0010 | 0.9990 |
| GradientBoosting | 0.9904 | 0.0010 | 0.9990 |
| RandomForest | 0.9899 | 0.0009 | 0.9991 |

XGBoost has the **lowest variance** (0.0003) and **highest stability score** — confirmed as final model.

### Step 3: Holdout Evaluation

The final model is evaluated on 10% of data that was **never seen during training or selection**:

```
Holdout Samples  : 2,860
Accuracy         : 99.41%
F1-Macro         : 99.40%
ROC-AUC          : 99.97%
MCC              : 98.80%
```
